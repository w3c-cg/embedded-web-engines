# Embedded Web Engines & Native Web Runtimes Community Group

This is the home repository of the W3C **Embedded Web Engines & Native Web Runtimes**
Community Group.

- Group page: https://www.w3.org/community/embedded-web-engines/
- Chair: Minori Ikeda

> **Note:** This is the work of a W3C *Community Group*. Documents published here
> are **not** W3C Standards nor are they on the W3C Standards Track. They do not
> represent the consensus or endorsement of the W3C Membership.

## Mission

> People reach more and more of the world's information — reading, learning,
> medical and scientific imagery, public services, transit and travel guidance,
> signage, connected devices and control surfaces — through software built on a
> web rendering engine running as a native application, rather than inside a
> traditional browser.

Our goal is to improve how web rendering engines are used in these settings, by
bringing together professionals from publishing, government services,
transportation, architecture, broadcasting, embedded devices, games, and
assistive technology.

## Participate

1. **Join the group** via the W3C group page above (a free W3C account is required).
   Joining means you agree to the
   [W3C Community Contributor License Agreement (CLA)](https://www.w3.org/community/about/process/cla/),
   which is required before making substantive contributions.
2. **Discuss** by opening or commenting on
   [issues](../../issues) in this repository. Asynchronous discussion in issues is
   our primary working channel, so that the record stays public and searchable.
   New here? Start by introducing yourself in the pinned
   [#1 Introductions](../../issues/1) issue.
   New to GitHub Issues? See [Communicating via GitHub Issues](./using-github-issues.md).
3. **Propose work** by opening an issue describing the problem before sending a
   pull request.
